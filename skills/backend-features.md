# Backend Features

## API Сервер (FastAPI)

### Основные эндпоинты

#### `/api/process-text`
**Метод:** POST  
**Описание:** Принимает распознанный текст голосовой команды, применяет Intent Guard и создаёт задачи при валидности.

**Тело запроса:**
```json
{
  "text": "Напишите отчёт по 5А классу к пятнице"
}
```

**Ответ:**
```json
{
  "valid": true,
  "transcript": "Напишите отчёт по 5А классу к пятнице",
  "tasks": [...],
  "count": 1
}
```

#### `/api/process-voice`
**Метод:** POST  
**Описание:** Принимает аудио файл, транскрибирует через Whisper, применяет Intent Guard и создаёт задачи.

**Тело запроса:** multipart/form-data с полем `audio`

#### `/api/tasks`
**Метод:** POST  
**Описание:** Создаёт задачу и отправляет уведомление исполнителю.

**Тело запроса:**
```json
{
  "description": "Подготовить отчёт",
  "assignee": "Иван Петрович",
  "due_date": "2024-01-15"
}
```

#### `/api/generate-document`
**Метод:** POST  
**Описание:** Генерирует официальное распоряжение директора на основе базы знаний (RAG).

**Тело запроса:**
```json
{
  "request": "Составить приказ о проверке учебников",
  "director_name": "Бекзат Сапаргалиевна",
  "match_count": 6
}
```

### Управление сотрудниками

#### `/api/staff`
**Метод:** GET  
**Описание:** Получает список всех сотрудников школы.

#### `/api/absences`
**Метод:** GET  
**Описание:** Получает заявки об отсутствии со статусом pending.

#### `/api/absences`
**Метод:** POST  
**Описание:** Создаёт заявку об отсутствии учителя.

**Тело запроса:**
```json
{
  "teacher_id": 123,
  "reason_text": "Болезнь"
}
```

#### `/api/absences/{absence_id}/cancel`
**Метод:** POST  
**Описание:** Отменяет заявку об отсутствии.

### Управление заменами

#### `/api/request-substitution`
**Метод:** POST  
**Описание:** Создаёт запись о замене со статусом 'pending' и отправляет push кандидату.

**Тело запроса:**
```json
{
  "absent_teacher_id": 123,
  "candidate_id": 456,
  "absence_id": 789,
  "lesson_number": 3,
  "class_name": "5А",
  "subject": "Математика",
  "room": "101",
  "reason": "Болезнь",
  "day_of_week": 2
}
```

#### `/api/substitution/{teacher_id}`
**Метод:** GET  
**Описание:** Находит кандидатов на замену для учителя.

**Параметры:**
- `teacher_id`: ID отсутствующего учителя
- `lesson_number`: номер урока
- `day_of_week`: день недели

#### `/api/substitutions/{substitution_id}`
**Метод:** DELETE  
**Описание:** Удаляет заявку на замену.

### Расписание

## Telegram Бот

### Основные команды

#### `/start`
**Описание:** Начало работы с ботом. Показывает приветствие и список команд.

#### `/report`
**Описание:** Отправить отчёт по столовой (только для учителей).
- Запрашивает текст отчёта
- NLP классификация сообщения
- Автоматическое сохранение в attendance_reports

#### `/help`
**Описание:** Показывает справку по командам бота.

### Функции бота

#### Обработка текстовых сообщений
- **Классификация:** Определяет тип сообщения (отчёт, заявка, замена)
- **Извлечение данных:** Вытаскивает класс, количество учеников, отсутствующих
- **Автосохранение:** Сохраняет отчёты в базу данных
- **Баллы:** Начисляет баллы за своевременные отчёты (до 09:00)

#### Обработка голосовых сообщений
- **Транскрибация:** Использует Groq Whisper для распознавания речи
- **NLP обработка:** Классифицирует и извлекает данные
- **Intent Guard:** Проверяет, является ли сообщение задачей
- **Создание задач:** Автоматически создаёт задачи при валидности

#### Управление заменами
- **Поиск кандидатов:** Автоматически находит доступных учителей
- **Push-уведомления:** Отправляет уведомления кандидатам с кнопками
- **Подтверждение:** Кандидат может принять или отклонить замену
- **Обновление статуса:** Автоматически обновляет статус замены

#### Уведомления
- **Задачи:** Отправляет уведомления о назначенных задачах
- **Замены:** Отправляет уведомления о предложенных заменах
- **Ежедневные отчёты:** Отправляет ежедневный отчёт по столовой

## Планировщик (APScheduler)

### Ежедневные задачи

#### `daily_canteen_report`
**Время:** 09:00 ежедневно  
**Описание:** Генерирует и отправляет ежедневный отчёт по столовой.

**Функции:**
- Собирает все отчёты за день
- Подсчитывает общее количество порций
- Отправляет отчёт сотрудникам столовой
- Отправляет отчёт директору
- Выделяет классы, которые не отправили отчёт

## NLP Обработка

### Классификация сообщений (`nlp_processor.py`)

#### `classify_message`
**Описание:** Классифицирует текстовое сообщение на основе контекста.

**Типы сообщений:**
- `attendance_report` - отчёт по посещаемости
- `absence_report` - заявка об отсутствии
- `task` - задача
- `other` - другое

#### `extract_attendance_data`
**Описание:** Извлекает данные из отчёта по посещаемости.

**Извлекаемые данные:**
- `class_name` - название класса
- `present_count` - количество присутствующих
- `absent_count` - количество отсутствующих
- `absent_list` - список отсутствующих

### Intent Guard

**Описание:** Строгий фильтр для голосовых команд директора.

**Алгоритм:**
1. Эвристическая проверка (стоп-фразы, глаголы действия)
2. LLM валидация через Groq
3. Проверка на конкретику поручения
4. Извлечение задач (description, assignee, due_date)

**Стоп-фразы:** приветствия, подтверждения, бессмысленные фразы  
**Глаголы действия:** напиши, подготовь, проверь, организуй, и т.д.

## Обработка голоса (`audio.py`)

### `transcribe_ogg`
**Описание:** Транскрибирует OGG/Opus (формат Telegram voice) в текст.

**Параметры:**
- `data`: байты аудио
- `language`: язык (по умолчанию "ru")

**Использует:** Groq Whisper API (whisper-large-v3)

## RAG Система (`rag_service.py`)

### `generate_official_document`
**Описание:** Генерирует официальное распоряжение директора на основе базы знаний.

**Процесс:**
1. Эмбеддинг запроса через Gemini
2. Поиск релевантных фрагментов в базе знаний (pgvector)
3. Генерация документа через Groq LLM
4. Форматирование в официально-деловой стиль

**Шаблон документа:**
```
# ПРИКАЗ
от [дата] № [номер]

## [Заголовок]

**ОСНОВАНИЕ:** [ссылки на приказы]

**ПРИКАЗЫВАЮ:**
1. [Пункт 1]
2. [Пункт 2]
3. Контроль за исполнением настоящего приказа оставляю за собой.

Директор школы _____________ / [имя директора]
```

## Логика замен (`logic.py`)

### `find_substitution`
**Описание:** Находит кандидатов на замену для отсутствующего учителя.

**Критерии отбора:**
- Свободен в указанный день и урок
- Преподает тот же предмет
- Имеет доступную нагрузку
- Не перегружен

### `generate_tomorrow_substitution_draft`
**Описание:** Генерирует черновик замен на завтра для всех отсутствующих.

### `auto_slot_task`
**Описание:** Автоматически создаёт задачи для замены на основе отсутствий.

## Генератор расписания (`scheduler_service.py`)

### `regenerate_schedule`
**Описание:** Генерирует расписание с нуля CP-SAT солвером.

**Параметры:**
- `class_names`: список классов
- `curriculum_overrides`: переопределения учебного плана
- `bands_input`: входные данные для бандов
- `days`: количество дней
- `periods_per_day`: количество уроков в день
- `time_limit_sec`: лимит времени решения
- `dry_run`: режим тестирования (без сохранения)

**Алгоритм:**
1. Создаёт переменные CP-SAT
2. Добавляет ограничения (конфликты учителей, кабинетов, классов)
3. Решает задачу оптимизации
4. Сохраняет результат в базу данных
