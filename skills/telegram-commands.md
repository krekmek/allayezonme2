# Telegram Bot Commands

## Обзор

Telegram бот - это основной канал взаимодействия учителей и сотрудников со школой. Бот поддерживает текстовые команды, голосовые сообщения и автоматическую обработку отчётов.

## Основные команды

### `/start`
**Описание:** Начало работы с ботом  
**Доступ:** Все пользователи

**Функции:**
- Регистрация пользователя в системе
- Отображение приветственного сообщения
- Показ списка доступных команд

**Пример ответа:**
```
👋 Добро пожаловать в Aqbobek Artificial Assistant!

Я помогу вам:
• Отправлять отчёты по столовой
• Сообщать об отсутствиях
• Управлять заменами

Доступные команды:
/report — отправить отчёт по столовой
/help — справка по командам
```

### `/report`
**Описание:** Отправить отчёт по столовой (только для учителей)  
**Доступ:** Учителя (role = "teacher")

**Процесс:**
1. Пользователь вводит команду `/report`
2. Бот запрашивает текст отчёта
3. Пользователь отправляет текст или голосовое сообщение
4. NLP классифицирует сообщение и извлекает данные
5. Бот сохраняет отчёт в базу данных
6. Бот отправляет подтверждение

**Пример использования:**
```
Пользователь: /report
Бот: 📝 Отправьте отчёт по посещаемости столовой
Пользователь: 5А - 25 бала, 2 ауырып калды: Алиса, Борис
Бот: ✅ Отчёт сохранён!
   Класс: 5А
   Присутствует: 25
   Отсутствует: 2
   Порции: 25
   ID: 123
```

**Формат отчёта:**
- `5А - 25 бала, 2 ауырып калды` - базовый формат
- `5А: 25 присутствуют, 2 отсутствуют` - альтернативный формат
- `5А, 25 человек, отсутствуют: Иванов, Петров` - другой формат

**NLP классификация:**
- Определяет класс из текста
- Извлекает количество присутствующих
- Извлекает количество отсутствующих
- Извлекает список отсутствующих (если есть)

**Баллы за своевременность:**
- Отчёт до 09:00: +1 балл
- Отчёт после 09:00: 0 баллов

### `/help`
**Описание:** Показывает справку по командам бота  
**Доступ:** Все пользователи

**Пример ответа:**
```
📚 Справка по командам:

/report — отправить отчёт по столовой (только учителя)
/help — эта справка

💡 Подсказки:
• Можно отправлять голосовые сообщения
• Отчёты до 09:00 дают баллы
• Для замены сообщите об отсутствии заранее
```

## Голосовые сообщения

### Обработка голосовых сообщений
**Описание:** Бот автоматически обрабатывает голосовые сообщения и транскрибирует их.

**Процесс:**
1. Пользователь отправляет голосовое сообщение
2. Бот скачивает аудио файл
3. Транскрибирует через Groq Whisper
4. Классифицирует через Intent Guard
5. Если это задача - создаёт её в базе данных
6. Если это отчёт - сохраняет как отчёт

**Пример:**
```
Пользователь: [голосовое сообщение] "Напишите отчёт по 5А классу"
Бот: ✅ Задача создана!
   Описание: Написать отчёт по 5А классу
   Статус: Новое
```

**Intent Guard:**
- Отсеивает случайные фразы
- Проверяет наличие конкретики
- Извлекает описание, исполнителя, срок

## NLP Обработка

### Классификация сообщений

**Типы сообщений:**
- `attendance_report` - отчёт по посещаемости
- `absence_report` - заявка об отсутствии
- `task` - задача
- `other` - другое

**Алгоритм классификации:**
1. Анализ ключевых слов
2. Проверка структуры сообщения
3. Использование LLM для сложных случаев

### Извлечение данных из отчётов

**Извлекаемые поля:**
- `class_name` - название класса (например, "5А", "10Б")
- `present_count` - количество присутствующих (число)
- `absent_count` - количество отсутствующих (число)
- `absent_list` - список отсутствующих (имена)

**Примеры:**
```
"5А - 25 бала, 2 ауырып калды"
→ class_name: "5А"
→ present_count: 25
→ absent_count: 2
→ absent_list: null

"5Б: 20 человек, отсутствуют Иванов, Петров"
→ class_name: "5Б"
→ present_count: 20
→ absent_count: 2
→ absent_list: ["Иванов", "Петров"]
```

## Управление заменами

### Создание замены
**Описание:** При отметке отсутствия учителя директор может создать замену.

**Процесс:**
1. Директор отмечает учителя отсутствующим в дашборде
2. Система находит кандидатов на замену
3. Директор выбирает кандидата
4. Система отправляет уведомление кандидату в Telegram

**Уведомление кандидату:**
```
🔔 Предложение замены

Отсутствующий: Иванов И.И.
Урок: 3, Математика
Класс: 5А
Кабинет: 101
Причина: Болезнь

Принять замену? [✅ Да] [❌ Нет]
```

### Подтверждение замены
**Описание:** Кандидат может принять или отклонить замену.

**Кнопки:**
- `✅ Принять` - принять замену
- `❌ Отклонить` - отклонить замену

**При принятии:**
- Статус замены меняется на "accepted"
- Расписание обновляется
- Уведомление отправляется директору

**При отклонении:**
- Статус замены меняется на "rejected"
- Директор уведомляется
- Можно выбрать другого кандидата

## Уведомления

### Уведомления о задачах
**Описание:** При назначении задачи сотрудник получает уведомление.

**Формат:**
```
📋 Новая задача

Описание: Подготовить отчёт по посещаемости
Срок: 2024-01-20
От: Директор

[✅ Принять] [📅 Отложить]
```

### Уведомления о заменах
**Описание:** Кандидат получает уведомление о предложенной замене.

**Формат:**
```
🔔 Предложение замены

Отсутствующий: Иванов И.И.
Урок: 3, Математика
Класс: 5А
Кабинет: 101

[✅ Принять] [❌ Отклонить]
```

### Ежедневные отчёты
**Описание:** Ежедневно в 09:00 отправляется отчёт по столовой.

**Получатели:**
- Сотрудники столовой
- Директор

**Формат для столовой:**
```
📊 Ежедневный отчёт по столовой

Всего отчётов: 15
Всего порций: 350

По классам:
• 1А: 25 порций (25 чел.)
• 1Б: 28 порций (28 чел.)
...
```

**Формат для директора:**
```
📊 Ежедневный отчёт по столовой

Всего отчётов: 15
Всего порций: 350

Не отправили отчёт:
• 2А
• 3Б

Пожалуйста, свяжитесь с классными руководителями.
```

## Ошибки и их обработка

### Общие ошибки

**"Вы не учитель"**
- Причина: Пользователь пытается использовать команду `/report`, но не является учителем
- Решение: Добавить пользователя в таблицу staff с role = "teacher"

**"Не удалось распознать голос"**
- Причина: Проблема с аудио файлом или Whisper API
- Решение: Проверить качество аудио, повторить попытку

**"ИИ не распознал задачу"**
- Причина: Intent Guard отклонил сообщение
- Решение: Уточнить поручение, добавить конкретику

### Ошибки NLP

**"Не удалось определить класс"**
- Причина: Название класса не найдено в тексте
- Решение: Указать класс явно (например, "5А")

**"Не удалось извлечь количество"**
- Причина: Число не найдено в тексте
- Решение: Указать количество явно

## Настройки бота

### Переменные окружения

**В `backend/.env`:**
```
TELEGRAM_BOT_TOKEN=your_bot_token
```

**Получение токена:**
1. Найдите @BotFather в Telegram
2. Отправьте `/newbot`
3. Следуйте инструкциям
4. Скопируйте токен

### Webhook

**Настройка вебхука:**
```bash
curl -X POST "https://api.telegram.org/bot{TOKEN}/setWebhook" \
  -d "url={YOUR_WEBHOOK_URL}"
```

## Лимиты и ограничения

### Ограничения Telegram
- Максимальный размер сообщения: 4096 символов
- Максимальный размер фото: 10 MB
- Максимальный размер видео: 50 MB
- Максимальный размер голосового: 50 MB

### Rate limiting
- 30 сообщений в секунду на группу
- 20 сообщений в минуту на чат

## Безопасность

### Проверка пользователей
- Только зарегистрированные пользователи могут использовать бота
- Проверка по telegram_id в таблице staff

### Шифрование
- Все сообщения шифруются Telegram
- Данные хранятся в защищённой базе данных

## Мониторинг

### Логи
- Все команды логируются
- Ошибки логируются с деталями
- Статистика использования

### Метрики
- Количество сообщений
- Количество отчётов
- Количество созданных задач
- Время ответа
