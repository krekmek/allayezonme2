# Frontend Features

## Обзор

Frontend - это веб-дашборд (Next.js 14) для директора школы, обеспечивающий централизованное управление всеми аспектами школы.

## Основные страницы

### 🏠 Главная страница (`/`)
**Компонент:** `app/page.tsx`

**Функции:**
- Мониторинг состояния школы в реальном времени
- Быстрый доступ к ключевым разделам
- Статистика по задачам, инцидентам, посещаемости
- Виджеты с важной информацией

**Виджеты:**
- Статистика задач (новые, в работе, выполнено)
- Последние инциденты
- Статистика посещаемости
- Быстрые действия

### 📋 Задачи (`/tasks`)
**Компонент:** `app/tasks/page.tsx`

**Функции:**
- Kanban-доска с задачами (Новое, В работе, Готово)
- Создание новых задач
- Редактирование существующих задач
- Перетаскивание задач между колонками
- Фильтрация по статусу и исполнителю

**Статусы задач:**
- `new` - Новые задачи
- `in_progress` - В работе
- `done` - Выполнено

**Поля задачи:**
- `description` - Описание задачи
- `assignee` - Исполнитель (ФИО)
- `due_date` - Срок выполнения
- `status` - Статус
- `source` - Источник (voice, text, telegram)

### 🚨 Инциденты (`/incidents`)
**Компонент:** `app/incidents/page.tsx`

**Функции:**
- Список всех инцидентов в школе
- Создание новых инцидентов
- Изменение статуса инцидента
- Фильтрация по статусу и типу

**Типы инцидентов:**
- Драка
- Пропуск урока
- Повреждение имущества
- Нарушение дисциплины
- Другое

**Статусы:**
- `new` - Новые
- `in_progress` - В работе
- `resolved` - Решённые

### 🍽️ Столовая (`/canteen`)
**Компонент:** `app/canteen/page.tsx`

**Функции:**
- Просмотр отчётов по посещаемости столовой
- Статистика по классам
- Генерация отчётов за период
- Выгрузка отчётов

**Данные отчёта:**
- `class_name` - Название класса
- `present_count` - Количество присутствующих
- `absent_count` - Количество отсутствующих
- `absent_list` - Список отсутствующих
- `portions` - Количество порций
- `created_at` - Время создания

### 📅 Расписание (`/schedule`)
**Компонент:** `app/schedule/page.tsx`

**Функции:**
- Просмотр расписания уроков
- Фильтрация по классу и учителю
- Редактирование расписания (drag & drop)
- Валидация изменений

**Поля урока:**
- `class_name` - Класс
- `lesson_number` - Номер урока
- `day_of_week` - День недели
- `teacher_id` - ID учителя
- `subject` - Предмет
- `room` - Кабинет

### 👥 Коллектив (`/staff`)
**Компонент:** `app/staff/page.tsx`

**Функции:**
- Список всех сотрудников
- Фильтрация по роли
- Просмотр информации о сотруднике
- Отметка отсутствия
- Создание заявки на замену

**Роли сотрудников:**
- `director` - Директор
- `admin` - Администратор
- `vice_director` - Замдиректора
- `teacher` - Учитель
- `maintenance` - Техперсонал
- `kitchen` - Столовая

**Информация о сотруднике:**
- `fio` - ФИО
- `specialization` - Специализация
- `role` - Роль
- `telegram_id` - Telegram ID
- `weekly_load` - Недельная нагрузка
- `max_load` - Максимальная нагрузка

### 📊 Общий график (`/staff-schedule`)
**Компонент:** `app/staff-schedule/page.tsx`

**Функции:**
- Просмотр общего графика сотрудников

- Проверка конфликтов

### 📚 База знаний (`/knowledge`)
**Компонент:** `app/knowledge/page.tsx`

**Функции:**
- Поиск по базе знаний приказов
- Просмотр выдержек из документов
- Генерация официальных распоряжений
- Упрощение сложных пунктов

**RAG функции:**
- Векторный поиск по документам
- Генерация документов на базе приказов
- Упрощение текста для учителей

### 🎓 Классы (`/classes`)
**Компонент:** `app/classes/page.tsx`

**Функции:**
- Список всех классов
- Просмотр расписания класса
- Статистика по классу

### ⚙️ Настройки (`/settings`)
**Компонент:** `app/settings/page.tsx`

**Функции:**

## API Routes

### `/api/voice/task`
**Метод:** POST  
**Описание:** Обрабатывает голосовые команды и текстовые задачи.

**Тело запроса:**
```json
{
  "text": "Напишите отчёт по 5А классу"
}
```

**Ответ:**
```json
{
  "valid": true,
  "transcript": "Напишите отчёт по 5А классу",
  "tasks": [...],
  "count": 1
}
```

### `/api/rag/simplify`
**Метод:** POST  
**Описание:** Упрощает сложный текст приказа.

**Тело запроса:**
```json
{
  "text": "Сложный юридический текст"
}
```

**Ответ:**
```json
{
  "bullets": [
    "Первый пункт упрощён",
    "Второй пункт упрощён",
    "Третий пункт упрощён"
  ]
}
```

### `/api/rag/generate-document`
**Метод:** POST  
**Описание:** Генерирует официальное распоряжение на базе знаний.

**Тело запроса:**
```json
{
  "request": "Составить приказ о проверке учебников",
  "director_name": "Бекзат Сапаргалиевна",
  "match_count": 6
}
```
